# TeacherPilot: The AI Command Center for Educators

**Eliminating decision fatigue for overwhelmed teachers**

> "The AI did 100 things. You see 3."

---

## What is this?

TeacherPilot is a daily command center built for a real teacher managing 272 students across 13 groups. The core problem it solves: teachers drown in administrative noise — grading backlogs, parent emails, lesson prep, attendance flags — spread across multiple tools with no coherent picture of what actually matters *right now*.

Instead of yet another chat interface that requires constant prompt engineering, TeacherPilot surfaces exactly **3 prioritized actions per day**, generated by Mistral AI after analyzing the teacher's live schedule, pending tasks, and deadlines. The teacher sees only what needs to happen next, not a firehose of everything.

The companion AI is named **Marimba** — a warm, emotionally intelligent assistant with the personality of a loyal border collie. Concise, proactive, and never overwhelming.

---

## How it works

```
Teacher's schedule + pending tasks
           |
           v
   Mistral Large 3 (AI reasoning)
           |
           v
  Top 3 prioritized actions
           |
           v
  Teacher sees 3 priority cards
           |
           v
   Tap a card → briefing + action
           |
           v
  Review, approve, mark done → next
```

The backend loads the teacher's rotating 6-day class schedule and all pending tasks (grading, communications, lesson prep). It sends this context to Mistral, which returns the 3 most relevant task IDs ranked by urgency, proximity to today's classes, and due dates. If no API key is configured, a local scoring algorithm takes over as fallback — so the app always works.

## Core User Flow

**Step 1 — Open**
Teacher opens TeacherPilot → Marimba's avatar greets them (text today, voice via ElevenLabs soon) → 3 **priority cards** are displayed, each showing class, subject, estimated time, and urgency at a glance.

**Step 2 — Tap**
Teacher taps a card → it expands into a full briefing: attendance flags, class history, AI-suggested actions. The card itself becomes the workspace — no navigation away.

**Step 3 — Marimba handles it**
Marimba surfaces the pre-drafted output — graded rubric, email draft, lesson plan — right inside the card. Teacher reviews, tweaks if needed, and approves. When voice is live, Marimba will also read the briefing aloud as the card opens.

**Step 4 — Voice (optional)**
Teacher speaks to Marimba's avatar at any time: *"What's urgent?"*, *"Help me prep 6B2"*, *"Mark that done."* Voxtral transcribes it, Mistral reasons about it, ElevenLabs speaks the response back. This is where the border collie personality becomes tangible, not just visual.

**Step 5 — Done**
Teacher marks the card done (tap or voice: *"Done"*) → card dismisses → next AI-ranked priority surfaces automatically → repeat until the day is handled.

> **Cards, not bullets.** Bullets were a list to read. Cards are a space to work in.

## Features

### Priority Cards (Home Screen)
Three AI-ranked cards shown at a time — no scrolling, no overwhelm. Each card displays class, subject, estimated time, and urgency. Dynamically generated by Mistral Large 3 based on today's schedule, deadlines, and pending workload.

### The Classroom Card
Tapping a priority card expands it into a full briefing: class history, absence flags, and AI-generated lesson suggestions — no extra navigation required.

### Communication Triage
Email tasks open a triaged view with AI-generated summaries and urgency tags. Urgent messages get auto-drafted responses for the teacher to review and approve.

### Marimba Voice Companion *(coming soon)*
An always-present avatar for optional voice interaction. Voice input is transcribed by Voxtral; spoken responses are generated via **ElevenLabs** (integration in progress).

## Getting Started

### Prerequisites
- Python 3.11+
- Node.js 18+ and npm

### Backend

```bash
cd backend
cp .env.example .env
# Optional: add your MISTRAL_API_KEY to .env for AI-powered priorities
pip install -r requirements.txt
python main.py
```

The server starts on `http://localhost:8000`:

| Endpoint | Description |
|---|---|
| `GET /api/priorities` | Top 3 AI-ranked tasks for today |
| `GET /api/schedule` | Teacher's class schedule |
| `GET /api/health` | Health check |
| `GET /api/docs` | Interactive API docs (Swagger UI) |

> **Without a Mistral key:** the endpoint still returns 3 tasks using a local scoring algorithm based on priority level, due dates, and today's schedule.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The app starts on `http://localhost:5173` and connects to the backend at `http://localhost:8000`.

### Running Tests

```bash
cd backend
pytest tests/ -v
```

## Environment Variables

Copy `backend/.env.example` to `backend/.env` and fill in values:

```
MISTRAL_API_KEY=your-mistral-api-key-here   # Required for AI prioritization
```

All other variables have safe defaults for local development.

## Security

- Never commit `.env` files — they are gitignored
- Use `.env.example` as the template (safe to commit, contains no secrets)
- All API keys are read from environment variables at runtime
- Report vulnerabilities responsibly before opening a public issue
