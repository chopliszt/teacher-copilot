# TeacherPilot: The AI Command Center for Educators

**Eliminating decision fatigue for overwhelmed teachers**

> "The AI did 100 things. You see 3."

## Context

Teachers face overwhelming administrative workloads and often have to manage multiple tabs to get through their day. Generic AI tools fall short because they require constant prompt engineering and lack persistent memory of a teacher's specific students, schedule, and context.

## General Idea

TeacherPilot is a daily command center designed to eliminate decision fatigue for overwhelmed educators. Operating under the vision "The AI did 100 things. You see 3," the system acts as a proactive assistant that triages and pre-drafts administrative tasks behind the scenes. Instead of a chat interface, the teacher interacts with **Marimba**—an emotionally connected, warm AI companion inspired by a loyal border collie.

## Core Features

### The 3 Bullets (Home Screen)
The application displays a maximum of three prioritized, one-sentence actions at a time. These priorities are dynamically generated by Mistral Large 3 by analyzing the teacher's schedule, deadlines, and pending communications.

### The Classroom Card
Tapping a class prep action immediately opens a single-screen briefing. This briefing includes class history, student absence flags, and AI-generated lesson suggestions without requiring extra clicks.

### Communication Triage
Email tasks open a triaged view featuring AI-generated summaries and urgency tags. For urgent messages, Marimba provides auto-drafted responses that the teacher simply needs to review and approve.

### Marimba Voice Companion
An always-present avatar allows for optional, natural voice interaction with a warm and concise persona. Voice input is transcribed by Voxtral, and spoken responses are generated via ElevenLabs.

## Tech Stack

- **Frontend**: ReactJS with TypeScript
- **Backend**: Python FastAPI
- **AI & APIs**: Mistral Large 3 (complex reasoning), Ministral (fast lookups), Voxtral (voice input), ElevenLabs (voice output)

## Security Best Practices

1. **Never commit sensitive data**: All environment variables and secrets should be in `.env` files which are gitignored.
2. **Use `.env.example`**: Copy `.env.example` to `.env` and fill in your actual values.
3. **Keep dependencies updated**: Regularly update your dependencies to patch security vulnerabilities.
4. **Use HTTPS**: Always use secure connections for API calls.
5. **Validate all inputs**: Sanitize and validate all user inputs to prevent injection attacks.
